OBS 반응 오버레이 사용 설명서
=============================


0. 먼저 알아둘 것
-----------------

  이 프로그램은 치지직 채팅에 특정 문구가 올라오면 OBS 화면 위에
  WebM, MP4, GIF, PNG 같은 오버레이를 띄우는 로컬 프로그램입니다.

  예시:

    채팅에 ? 입력      -> 샘플 오버레이 표시
    채팅에 ㅠㅠ 입력   -> 다른 오버레이 표시

  프로그램은 외부 서버가 아니라 방송하는 PC 안에서만 실행됩니다.
  주소가 127.0.0.1로 시작하는 것은 내 컴퓨터 안에서만 접속한다는 뜻입니다.

  중요:

    - 프로그램 폴더 전체를 한 곳에 압축 해제해서 사용하세요.
    - 폴더 안의 파일을 따로따로 옮기지 마세요.
    - 서버 실행 중에는 검은색 실행 창을 닫지 마세요.


1. 실행 방법
------------

  1) 압축을 풉니다.

     예시:

       바탕화면\OBS_Reaction_Overlay
       문서\OBS_Reaction_Overlay

     어느 위치에 풀어도 됩니다.

  2) 폴더 안의 아래 파일을 더블클릭합니다.

       OBS_Reaction_Overlay.exe

  3) 잠시 후 컨트롤 페이지가 자동으로 열립니다.

       http://127.0.0.1:39291/control

  4) 검은색 실행 창은 서버 창입니다.

     방송 중에는 이 창을 닫지 마세요.
     이 창을 닫으면 오버레이와 채팅 연동도 같이 멈춥니다.

  5) 컨트롤 페이지가 자동으로 안 열리면 아래 파일을 더블클릭하세요.

       Open_Control_Page.bat


2. OBS 기본 설정
----------------

  1) OBS를 엽니다.

  2) 소스 목록에서 + 버튼을 누릅니다.

  3) 브라우저를 선택합니다.

  4) URL에 아래 주소를 입력합니다.

       http://127.0.0.1:39291/overlay

  5) 브라우저 소스의 너비와 높이를 VSeeFace 소스 범위와 비슷하게 맞춥니다.

     이 브라우저 소스의 사각형 영역 안에서 오버레이가 랜덤으로 등장합니다.

  6) VSeeFace 소스와 오버레이 브라우저 소스를 함께 선택합니다.

  7) 우클릭 후 선택한 항목 그룹화를 누릅니다.

  8) 그룹 안에서 오버레이 브라우저 소스가 VSeeFace 소스보다 위에 오도록 배치합니다.

  9) 컨트롤 페이지에서 입력값 테스트를 눌러 오버레이가 보이는지 확인합니다.


3. 치지직 연동 방법
-------------------

  실제 채팅을 읽으려면 치지직 Developers에서 애플리케이션을 등록해야 합니다.
  Client ID와 Client Secret은 애플리케이션 등록 후 확인할 수 있습니다.

  1) 치지직 Developers에 접속합니다.

       https://developers.chzzk.naver.com

  2) 애플리케이션을 등록합니다.

     이름은 자유롭게 정해도 됩니다.
     단, chzzk, 치지직, naver, 네이버 같은 공식 서비스명은 넣지 않는 편이 안전합니다.

  3) 로그인 리디렉션 URL에는 아래 값을 입력합니다.

       http://127.0.0.1:39291/auth/chzzk/callback

  4) 권한은 채팅 메시지 조회를 선택합니다.

  5) 등록이 끝나면 Client ID와 Client Secret을 확인합니다.

  6) 컨트롤 페이지의 치지직 앱 정보에 Client ID와 Client Secret을 입력합니다.

  7) 앱 정보 저장을 누릅니다.

     저장 후 화면에서는 Client ID와 Client Secret이 안 보이도록 마스킹됩니다.

  8) 치지직 로그인을 누릅니다.

     브라우저에서 치지직 권한 동의 화면이 열립니다.
     동의가 끝나면 로그인 정보가 이 PC에 저장됩니다.

  9) 컨트롤 페이지로 돌아와서 채팅 연결 시작을 누릅니다.

  10) 상태에 아래처럼 보이면 준비 완료입니다.

        연결: 연결됨
        구독: 완료

  11) 치지직 채팅에 등록한 반응 문구를 입력해서 오버레이가 뜨는지 확인합니다.


4. 컨트롤 페이지 설명
--------------------

  4-1. 치지직 앱 정보

    Client ID:
      치지직 Developers에서 발급받은 Client ID입니다.
      저장 후에는 화면에 보이지 않도록 마스킹됩니다.

    Client Secret:
      치지직 Developers에서 발급받은 Secret입니다.
      비밀번호처럼 취급하세요.
      저장 후에는 화면에 보이지 않도록 마스킹됩니다.

    앱 정보 저장:
      Client ID와 Client Secret을 이 PC에 저장합니다.

    치지직 로그인:
      치지직 권한 동의 화면을 엽니다.

    채팅 연결 시작:
      실제 채팅 감지를 시작합니다.

    재연결:
      연결 상태가 이상하거나 채팅이 안 들어오는 것 같을 때 다시 연결합니다.

    연결 중지:
      채팅 감지를 중지합니다.


  4-2. 연결 상태

    앱 정보:
      Client ID와 Client Secret이 저장되어 있는지 표시합니다.

    로그인:
      치지직 로그인이 완료되었는지 표시합니다.

    연결:
      치지직 채팅 서버와 연결되어 있는지 표시합니다.

    구독:
      채팅 이벤트를 받을 준비가 되었는지 표시합니다.
      구독 완료가 되어야 실제 채팅을 받을 수 있습니다.

    최근 채팅:
      프로그램이 마지막으로 받은 채팅입니다.

    마지막 채팅:
      실제 채팅을 마지막으로 받은 시간입니다.

    마지막 heartbeat:
      치지직 채팅 서버와 연결이 살아 있는지 확인한 마지막 시간입니다.
      채팅이 없어도 이 시간이 주기적으로 갱신되면 연결이 유지되고 있는 상태입니다.

    재연결 횟수:
      실제 연결 오류로 자동 재연결이 일어난 횟수입니다.

    마지막 재연결:
      가장 최근에 자동 재연결이 일어난 시간입니다.

    재연결 사유:
      마지막 자동 재연결이 왜 일어났는지 표시합니다.


  4-3. 표시 방식

    랜덤 위치:
      OBS 브라우저 소스 영역 안에서 랜덤 위치에 오버레이가 표시됩니다.

    고정 위치:
      X 좌표와 Y 좌표 위치에만 오버레이가 표시됩니다.

    너비, 높이:
      오버레이 하나가 화면에 표시되는 크기입니다.

    가장자리 여백:
      오버레이가 영역 가장자리에 너무 붙지 않도록 하는 여백입니다.

    볼륨(%):
      오버레이 동영상 소리 크기입니다.
      0은 음소거, 100은 원본 크기입니다.

    한번에 보이는 개수:
      동시에 화면에 남겨둘 오버레이 개수입니다.

      예시:

        1로 설정:
          새 반응이 들어오면 기존 오버레이를 끊고 새 오버레이를 재생합니다.

        3으로 설정:
          최대 3개까지 동시에 표시합니다.
          네 번째 반응이 들어오면 가장 오래된 오버레이를 제거합니다.


  4-4. 채팅 시뮬레이션

    실제 치지직 채팅을 보내지 않고도 테스트할 수 있습니다.

    예시:

      테스트 채팅에 ? 입력
      입력값 테스트 클릭
      샘플 오버레이가 OBS에 뜨는지 확인


  4-5. 반응 오버레이

    오버레이명:
      관리용 이름입니다.

    오버레이 이미지/동영상 파일명:
      assets 폴더 안의 파일명입니다.

    파일 업로드:
      새 오버레이 파일을 assets 폴더에 복사합니다.

    반응 채팅:
      어떤 채팅에 반응할지 정합니다.
      한 항목에 한 문구씩 추가합니다.
      채팅 내용과 정확히 일치할 때만 반응합니다.

    예시:

      ?
      ??
      ???
      ㅠㅠ
      ㅋㅋ

    이 오버레이 사용:
      체크되어 있는 오버레이만 실제로 반응합니다.

    지원 파일 형식:

      svg, webm, mp4, mov, gif, png, jpg, jpeg, webp


5. 연결 구조
------------

  채팅이 없다는 이유만으로 재연결하지 않습니다.

  소켓 읽기 타임아웃은 서버가 멈췄는지 확인하기 위한 짧은 점검 주기입니다.
  채팅이 없는 시간에는 이 타임아웃이 발생해도 연결을 끊지 않습니다.

  치지직 채팅 연결은 Socket.IO / Engine.IO 방식입니다.
  이 프로그램은 일정 시간마다 heartbeat를 보내고, 서버 응답을 받아 연결 상태를 확인합니다.

  재연결은 아래 상황에서만 일어납니다.

    - 치지직 채팅 서버 연결이 실제로 끊김
    - 네트워크 오류가 발생함
    - 치지직 API 요청이 실패함
    - 3분 이상 소켓 응답이 전혀 없어 연결이 멈춘 것으로 판단됨
    - 사용자가 재연결 버튼을 누름

  자동 재연결은 2초부터 시작해서 실패할수록 대기 시간이 늘어납니다.
  최대 대기 시간은 8초입니다.


6. 폴더와 파일 설명
-------------------

  OBS_Reaction_Overlay.exe
    서버를 실행하는 파일입니다.
    이 파일을 더블클릭해서 시작합니다.

  Open_Control_Page.bat
    컨트롤 페이지를 여는 파일입니다.
    서버가 이미 켜져 있을 때 사용합니다.

  Make_Distribution_Zip.bat
    다른 사람에게 보낼 ZIP 파일을 만드는 파일입니다.
    credentials.json, chzzk_tokens.json, chzzk_auth_state.json은 자동으로 제외합니다.

  config.json
    오버레이 설정이 저장됩니다.

  credentials.json
    Client ID와 Client Secret이 저장됩니다.
    처음에는 없어도 됩니다.
    앱 정보 저장을 누르면 자동으로 생성됩니다.

  chzzk_tokens.json
    치지직 로그인 정보가 저장됩니다.
    처음에는 없어도 됩니다.
    치지직 로그인을 완료하면 자동으로 생성됩니다.

  assets 폴더
    오버레이 이미지와 동영상 파일이 저장됩니다.

  public 폴더
    컨트롤 페이지와 OBS 오버레이 페이지 파일이 들어 있습니다.

  THIRD_PARTY_LICENSES.txt
    이 프로그램에 포함된 오픈소스 구성요소와 라이선스 요약입니다.

  LICENSE
    이 프로그램 자체의 오픈소스 라이선스입니다.


7. 다른 사람에게 공유할 때
--------------------------

  압축해서 공유할 때 아래 파일은 빼는 편이 안전합니다.

    credentials.json
    chzzk_tokens.json
    chzzk_auth_state.json

  이 파일들은 사용자마다 다릅니다.
  받는 사람이 프로그램을 설정하면 자동으로 다시 생성됩니다.

  공유할 때 포함해야 하는 것:

    OBS_Reaction_Overlay.exe
    Open_Control_Page.bat
    config.json
    assets 폴더
    public 폴더
    README.txt
    LICENSE
    THIRD_PARTY_LICENSES.txt

  가장 쉬운 공유 방법:

    0) Build_Windows_Exe.bat을 실행해서 exe를 먼저 만듭니다.
    1) Make_Distribution_Zip.bat을 더블클릭합니다.
    2) OBS_Reaction_Overlay_Package.zip 파일이 생성됩니다.
    3) 생성된 ZIP 파일을 공유합니다.


8. 문제가 생겼을 때
-------------------

  컨트롤 페이지가 안 열릴 때:

    - Start_Overlay_Server.bat을 다시 더블클릭합니다.
    - 또는 OBS_Reaction_Overlay.exe를 다시 더블클릭합니다.
    - 검은색 서버 창이 열려 있는지 확인합니다.

  OBS에 오버레이가 안 보일 때:

    - OBS 브라우저 소스 URL이 맞는지 확인합니다.
    - URL은 http://127.0.0.1:39291/overlay 입니다.
    - 컨트롤 페이지에서 입력값 테스트를 먼저 해봅니다.

  채팅이 안 들어올 때:

    - 상태에서 연결: 연결됨, 구독: 완료인지 확인합니다.
    - 아니면 재연결을 누릅니다.
    - 그래도 안 되면 치지직 로그인을 다시 진행합니다.
